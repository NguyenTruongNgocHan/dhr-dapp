# dhr-dapp Contract

## 1. Overview
This folder contains the smart contract layer of dhr-dapp.

The main contract is written in Vyper and is responsible for:
- patient registration
- doctor registration
- access permission control (grant/revoke)
- health record management

The contract is compiled and deployed using Ape Framework to a local Geth blockchain.

## 2. Folder contents
- `contracts/DHR.vy`: Main Vyper smart contract.
- `scripts/deploy.py`: Deployment script used by Ape.

## 3. Requirements
Before compiling or deploying, make sure:
- Ape Framework is installed.
- Local Geth node is running at `http://127.0.0.1:8545`.
- You are in this `contract` folder.

## 4. Compile contract
Run:

```bash
ape compile
```

This generates build artifacts, including ABI/build output.

## 5. Deploy contract
Run:

```bash
ape run deploy --network ethereum:local:http://127.0.0.1:8545
```

This executes `scripts/deploy.py` and deploys the contract to the local network.

## 6. Output after deployment
After deployment, you should get the deployed contract address in the console output.

Important:
- Save the contract address.
- Keep the ABI/build output from the compile/build artifacts.

## 7. Notes for backend integration
For backend integration (FastAPI + Web3.py):
- Use the deployed contract address from deployment output.
- Use the ABI/build output generated by Ape.
- Update backend contract configuration after each new deployment.
